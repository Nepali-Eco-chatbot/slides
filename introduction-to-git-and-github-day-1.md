## How do multiple developers work on the same project at once? 
<!-- stop -->
Developers working on the same project often have to generate different version of code from a common point.
> For example: [diagram](./images/multiple-version-for-one-project.png) [demo](https://excalidraw.com/#json=_tRfCbHPLWS1OHnI2Igwv,9MbNW6moCcPjR6Y62SOEgQ)

<!-- stop -->
So, to solve this problem there are tools and software know as VCS(Version control System) or SCM(Source Control Management).

<!-- stop -->
And git is one of the most popular distributed scm tool. 

---
## History of git.
<!-- stop -->
Since 2002 Linux kernel was using [BitKeeper](https://en.wikipedia.org/wiki/BitKeeper) as their SCM system. But one of the developers named [Andrew Tridgell](https://en.wikipedia.org/wiki/Andrew_Tridgell) reversed engineered bit keeper and created a bit keeper client know as SourcePuller. Which lead their free licence being revoked and there were no other good free alternatives. That is why [linus](https://en.wikipedia.org/wiki/Linus_Torvalds) started the Git project.

You can read in detail what happend at that time [here](https://lwn.net/Articles/130746/#Comments)

## So, what is github? 
<!-- stop -->
A cloud based platform for hosting git source code to collaborate and share code with others. 
<!-- stop -->
It provides collaborative and convinence tools on top of git. It was launched on 2008 and brought for around 7.5 Billion by microsoft in 2018.
